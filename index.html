The goal of robota is to primarily make using web workers more powerful, and easier. There are some common issues and scenarios that robota has been desgined to address.

  • To be able to pass functions in-line if desired, rather than in external .js libs.
  • To flexibly use the maximum number of processors that your browsers host features.
  • To run a batch of x tasks, triggering a callback on each tasks completion.
  • To run a batch of x tasks, triggering a single callback when all are complete.
  • To run a chain of tasks, where the resuts of one is fed to the next, triggering a callback when all are complete.


In this secenario we need to dispatch an arbitrary number of independent work units. In this case, we do not care when each unit completes, we only care how quickly ALL of the work units are completed. Optionally we may want a callback to be executed as each unit completes.

A usage scenario would be a user creating 3000 non-instanced spheres in a WebGL library. Scene drawing cannot go ahead until all of the spheres have been added to the scene graph.

Here we need to dispatch an arbitrary number of work units. Once ALL units are complete, we need to receive the results back via a callback. The results should be in the same order in which they were dispatched.

For our usage scenario, we will take an array of data and each unit will be multiplied by a fixed value. Once all units are complete we will multiply the first two results, and then alternatively divide and multiply the current total. Key to this scenario is that the end result is always the same.
